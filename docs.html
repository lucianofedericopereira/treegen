Two steps

Quick start

1 — Drop a placeholder anywhere in your README.md:

## Project structure

[[files]]

2 — Add a workflow at .github/workflows/update-tree.yml:

name: Update file tree
on:
  push:
    branches: [main]
permissions:
  contents: write
jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: lucianofedericopereira/treegen@v1
        with:
          style: ascii    # ascii | svg | collapsible
Prefer no placeholder? Write the managed markers yourself and treegen will keep the region between them in sync: 
<!-- filetree:start dir="src" style="svg" -->
<!-- filetree:end -->
Authoring

Markers & placeholders

Two ways to mark where a tree goes:

  • Placeholder — a one-off token, [[files]], optionally with attributes. On first run it expands into a managed block.
  • Marker block — a <!-- filetree:start … --> / <!-- filetree:end --> pair. Everything between is regenerated every run; the start line's attributes are preserved.

Attributes live on the marker, so one file can render many directories:

[[files dir="server" style="ascii"]]
[[files dir="web/src" style="svg" color="green"]]
[[files dir="deploy" style="collapsible" max-depth="2"]]
Fence-safe. Markers and [[…]] tokens inside fenced code blocks or `inline code` are left untouched — so you can document the syntax in the very same file it edits.

Per-marker attributes

Any marker overrides the workflow defaults inline.

AttributeExampleMeaning

stylestyle="svg"ascii, svg, or collapsible.
colorcolor="green"SVG folder colour (see palette above).
max-depthmax-depth="2"Limit depth (0 = unlimited).
dirs-onlydirs-only="true"Directories only, hide files.

svg-outputsvg="docs/t.svg"Where the SVG file is written.
collapsecollapse="true"Wrap the block in one top-level <details>.

Beyond Markdown

HTML output

On .html files (or with format: html) the same markers emit styled HTML — a <pre>, an <img>, or a <details> tree. This very page is built that way. 

python -m treegen --readme index.html --format html --no-placeholder
  • --format html (or auto) emits HTML.
  • --no-placeholder regenerates only explicit markers, leaving literal [[files]] docs alone.
Ship it

CI, Pages & Marketplace

Keep PRs honest

Fail a pull request when the README is stale instead of committing:

- uses: lucianofedericopereira/treegen@v1
  with:
    check: "true"
    commit: "false"

GitHub Pages

Ship index.html + .nojekyll, then Settings → Pages → Deploy from a branch → main / / (root).

Marketplace

Push public, create a release tagged v1.0.0, tick “Publish this Action to the Marketplace”, then float a v1 tag:

git tag -fa v1 -m "treegen v1" && git push origin v1 --force
Local

Command line

Standard library only — nothing to install.

python -m treegen --readme README.md --style svg --color green --dir src
python -m treegen --readme README.md --check      # preview, don't write

Run python -m treegen --help for every flag.
